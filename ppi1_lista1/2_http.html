Protocolo HTTP

O que é?

HTTP, ou Hypertext Transfer Protocol é um protocolo de comunicação utilizado para a transferência de informações na World Wide Web(WWW) e em outros sistemas de rede. Ele serve como a base para que um cliente e um servidor web troquem informações.

Como é usado na WEB

Esse protocolo HTTP é o motor da World Wide Web (Web) e é usado para permitir que o navegador acesse e visualize qualquer conteúdo de um servidor web. A sua utilização na Web se baseia em um modelo constante de Requisição-Resposta dentro da arquitetura Cliente-Servidor.

Campos da Requisição(Request) HTTP

A Requisição é a mensagem que o cliente (navegador) envia ao servidor para solicitar um recurso na Web.

  1. Linha de Status: É a primeira linha e informa o resultado da requisição.
    • Inclui a Versão do HTTP e o Código de Status (um número de três dígitos).
    • O código 200 OK indica que a requisição foi bem-sucedida, enquanto o código 404 Not Found indica que o recurso não foi localizado.
  2. Cabeçalhos (Headers): Fornecem metadados e informações sobre o cliente.
    • O campo Host é crucial, identificando o domínio do servidor de destino.
    • O User-Agent informa o servidor sobre qual navegador (e sistema operacional) está fazendo a solicitação.
    • O campo Accept indica os tipos de conteúdo que o cliente é capaz de processar (ex: HTML, imagens).
  3. Corpo (Body):
    • Geralmente fica vazio em requisições GET.
    • É preenchido com o payload (dados do formulário, login, etc.) nas requisições POST.

Campos da Resposta (Response) HTTP

A Resposta é a mensagem que o servidor envia de volta, fornecendo o resultado da solicitação do cliente.

  1. Linha de Status: É a primeira linha e informa o resultado da requisição.
    • Inclui a Versão do HTTP e o Código de Status (um número de três dígitos).
    • O código 200 OK indica que a requisição foi bem-sucedida, enquanto o código 404 Not Found indica que o recurso não foi localizado.
  2. Cabeçalhos (Headers): Fornecem metadados sobre o servidor e sobre o conteúdo que está sendo enviado.
    • O campo Server identifica o software do servidor web.
    • O Content-Type especifica o formato do conteúdo no corpo, que é comumente o HTML (ex: text/html).
    • O Content-Length indica o tamanho do corpo da mensagem.
  3. Corpo (Body):
    • É o conteúdo principal da resposta.
    • Para a Web, este corpo comumente contém o código HTML, que será processado pelo navegador para exibir a página.
